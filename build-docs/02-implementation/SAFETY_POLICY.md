## üõ°Ô∏è Safety Filters Explained

### What Gets Filtered (Public Generator)

The public generator rejects workflow requests containing:

1. **NSFW Content**
   - Adult/explicit material
   - Sexual services or dating app exploitation
   - Anything inappropriate for workplace environments

2. **Illegal Activities**
   - Hacking, cracking, or exploiting systems
   - Credential stuffing or brute force attacks
   - Fraud, identity theft, or financial crimes
   - Copyright infringement or DRM bypass
   - Academic dishonesty (exam cheating, plagiarism)

3. **Privacy Violations**
   - Scraping personal data without consent
   - Surveillance or stalking capabilities
   - Unauthorized location tracking
   - Email/phone harvesting for spam

4. **Terms of Service Violations**
   - Automated account creation
   - Fake engagement or review manipulation
   - Bypassing paywalls without authorization
   - Spam or bot creation

## üîß Customizing for Private Use

If you're running your own instance and need to modify safety filters:

### 1. Edit Safety Keywords
```python
# scripts/workflow_generator.py

class KimiWorkflowGenerator:
    # Customize these lists for your use case
    NSFW_KEYWORDS = [...]
    ILLEGAL_KEYWORDS = [...]
    PRIVACY_KEYWORDS = [...]
```

### 2. Disable Safety Checks (Advanced)
```python
def check_safety(self, use_case: str, industry: Optional[str] = None):
    # For private instances, always return safe
    return {'safe': True}
```

### 3. Environment Variable
```bash
# Set this to disable public safety filters
export BROWSEROS_DISABLE_SAFETY_FILTERS=true
```

**IMPORTANT:** If you disable safety filters, you assume full legal responsibility for workflows generated. Ensure your use complies with:
- Local, state, and federal laws
- Website Terms of Service
- Data privacy regulations (GDPR, CCPA, etc.)
- Ethical guidelines for your industry
